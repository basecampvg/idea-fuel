## Technical SEO Fixes

### Immediate (Week 1)

1. **Verify Google Search Console** — Submit sitemap, check indexing status
2. **Check if glossary/docs are actually indexed** — 250 glossary pages should be generating impressions. If they're not indexed, there's a crawling/rendering issue.
3. **Add internal linking structure** — Every glossary term should link to relevant /learn/, /ideas/, and /guides/ pages. Every blog post should link to 3-5 glossary terms.
4. **Verify structured data** — Test all JSON-LD with Google's Rich Results Test
5. **Check page speed** — `force-dynamic` on the sitemap is fine, but make sure content pages use ISR/SSG, not SSR

### Ongoing

6. **Build an internal linking component** — Automatically link glossary terms when they appear in any content page
7. **Add breadcrumbs** — BreadcrumbList schema on all content pages
8. **Create topic cluster pages** — Hub pages at `/learn/`, `/ideas/`, `/tools/`, `/guides/` that link to all children
9. **Canonical URLs** — Ensure no duplicate content between /learn/ and /docs/ for overlapping topics

## Content Production System

### Automated Pipeline (Build This)

IdeaFuel already has AI content generation infrastructure. Build a content pipeline:

```
1. Keyword Research (DataForSEO API)
   → Identify target keyword + related long-tails
   
2. Content Generation (Claude/GPT)
   → Generate draft in Mark Manson x Peter Thiel voice
   → Include IdeaFuel validation data where relevant
   
3. Content Review (Matt)
   → Edit for quality, add personal takes
   → 15-30 min per post
   
4. Publish (Admin API)
   → Blog admin already supports CRUD
   → Extend to /ideas/, /learn/, /guides/ content types
   
5. Internal Linking (Automated)
   → Auto-link glossary terms in all content
   → Add related content suggestions
```

### Content Database Schema Extension

Add tables for programmatic content:
- `seo_pages` — stores /ideas/, /learn/, /guides/ content (similar to blogPosts)
- `seo_page_type` — enum: IDEAS_LIST, LEARN, GUIDE, TOOL
- Include fields: target_keyword, secondary_keywords, search_volume, current_rank

This lets you track SEO performance per page and prioritize updates.
